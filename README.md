# Telegram-бот: реєстр корупційних правопорушень НАЗК

Бот для пошуку та аналізу інформації в офіційному реєстрі корупційних правопорушень НАЗК. Працює з API НАЗК та (опційно) з Google Gemini для текстового висновку про корупційні ризики.

Усі дії доступні **через команди** та **через reply-меню** (кнопки внизу екрану).

---

## Команди бота

| Команда | Опис |
|--------|------|
| **/start** | Привітання, список команд та показ меню (reply-клавіатура). |
| **/help** | Детальний опис можливостей та формату введення ПІБ. |
| **/stop** | Скасування поточної операції (наприклад, введення ПІБ для пошуку чи аналізу). |
| **/search** | Пошук особи в реєстрі за ПІБ. |
| **/analyze** | Короткий висновок про корупційні ризики особи (AI, Gemini). |
| **/load_all** | Завантажити весь реєстр одним JSON-файлом. |
| **/revelio** | Випадкова особа з уже завантаженого реєстру (для тесту). |

---

## Reply-меню (кнопки)

Після `/start` з’являються кнопки:

- **🔍 Пошук** — той самий сценарій, що й `/search`.
- **🤖 Аналіз (AI)** — той самий сценарій, що й `/analyze`.
- **📥 Завантажити реєстр (JSON)** — той самий сценарій, що й `/load_all`.
- **📋 Переглянути довідку** — той самий текст, що й `/help`.

Під час очікування ПІБ (після `/search` або «Пошук» / «Аналіз») натискання будь-якої кнопки меню скасовує поточний ввід і виконує обрану дію.

---

### /start

Показує привітання, перелік команд та reply-клавіатуру з кнопками (Пошук, Аналіз, Завантажити реєстр, Довідка).

---

### /help

Опис можливостей бота, формат ПІБ (Прізвище Ім'я По батькові), контакт розробника та посилання на НАЗК. Після відповіді знову показується меню.

---

### /stop

Скидає FSM-стан і відправляє «Операцію скасовано.» Зручно, коли бот чекає введення ПІБ, а користувач передумав.

---

### /search

**Що робить:** пошук особи в реєстрі НАЗК за повним ПІБ.

1. Натисни `/search` або кнопку **🔍 Пошук**.
2. Введи ПІБ у форматі **Прізвище Ім'я По батькові** (три частини через пробіл), наприклад: `Петренко Петро Петрович`.
3. Бот покаже: «Шукаю інформацію про: … Зачекай кілька секунд.»
4. Після відповіді API те саме повідомлення буде відредаговано: замість «Шукаю…» з’явиться результат пошуку (або повідомлення про відсутність даних).

**Результат:** блок з даними з реєстру (тип суб’єкта, покарання, суд, справи, статті кодексу тощо). Показуються до 3 записів; якщо їх більше — примітка «Показано перші 3 запис(и) з N».

---

### /load_all

**Що робить:** завантажує всі записи реєстру через API НАЗК і надсилає їх у вигляді одного JSON-файлу в чат.

1. Натисни `/load_all` або кнопку **📥 Завантажити реєстр (JSON)**.
2. З’явиться повідомлення з попередженням і **інлайн-кнопками** «Підтвердити» / «Скасувати».
3. Після підтвердження повідомлення оновлюється: «Завантаження реєстру... минуло 0 сек.»; кожні **5 секунд** текст змінюється (5 сек., 10 сек. тощо).
4. Після завершення — «Завантаження завершено. Готую файл...», потім бот надсилає файл `corrupt_all_data.json`.

**Підпис файлу:** «Файл із усіма записами реєстру корупційних правопорушень. Час формування: X хвилин Y секунд.»

Файл зберігається в корені проєкту і використовується командою `/analyze`.

---

### /analyze

**Що робить:** формує **короткий текстовий висновок** про корупційні ризики особи природною мовою (українською). Використовує дані з файлу реєстру та Google Gemini.

1. Натисни `/analyze` або кнопку **🤖 Аналіз (AI)**.
2. Введи ПІБ у форматі **Прізвище Ім'я По батькові**.
3. Бот перевіряє файл `corrupt_all_data.json` у корені проєкту (його можна отримати командою `/load_all`).
4. **Якщо особи немає в реєстрі** — повертається повідомлення про відсутність даних; запит до Gemini не виконується.
5. **Якщо особа є в реєстрі** — дані передаються в Gemini, і бот надсилає висновок з урахуванням давності, суду та рекомендацією щодо видачі позики.

**Вимога:** у `.env` має бути заданий `GEMINI_API_KEY` (безкоштовний рівень Google AI Studio). Якщо ключа немає, при знайденій особі бот повідомить про необхідність налаштування ключа.

---

### /revelio

Повертає випадкове ПІБ з уже завантаженого реєстру (`corrupt_all_data.json`). Зручно для перевірки пошуку чи аналізу. Якщо файлу немає — бот попросить спочатку виконати `/load_all`.

```
│   │   ├── reply.py        # get_main_reply_keyboard (Пошук, Аналіз, Завантажити реєстр, Довідка)
│   │   └── inline.py       # get_load_all_confirmation_keyboard (Підтвердити / Скасувати)

│       ├── corrupt_api.py   # клієнт API НАЗК (find, get_all)

├── .env                    # не комітити: BOT_TOKEN, GEMINI_API_KEY, опційно API_BASE_URL та endpoints
├── corrupt_all_data.json   # з’являється після /load_all у корені проєкту

```

## Встановлення та запуск

### 1. Залежності

```bash
python -m venv .venv
.venv\Scripts\activate   # Windows
pip install -r requirements.txt
```

### 2. Налаштування `.env`

У корені проєкту створіть файл `.env`:

```env
BOT_TOKEN=токен_від_BotFather
GEMINI_API_KEY=ключ_з_Google_AI_Studio
```

- **BOT_TOKEN** — обов’язково для роботи бота.
- **GEMINI_API_KEY** — потрібен лише для команди `/analyze` (безкоштовний рівень достатній).

Опційно (значення за замовчуванням вказані в `bot/config.py`):

- **API_BASE_URL** — базовий URL API НАЗК.
- **CORRUPT_FIND_ENDPOINT**, **CORRUPT_GET_ALL_ENDPOINT** — endpoints для пошуку та завантаження всього реєстру.

### 3. Запуск

Запускати **обов’язково з кореня проєкту**, щоб пакет `bot` імпортувався коректно:

```bash
python run_bot.py
```

або:

```bash
python -m bot.bot
```

Після запуску бот відповідає на команди та кнопки меню в Telegram.

## Коротко

- **/start** — привітання, список команд, показ reply-меню.  
- **/help** — опис можливостей та формат ПІБ; після відповіді знову показується меню.  
- **/stop** — скасування поточної операції (введення ПІБ).  
- **/search** — пошук за ПІБ; результат підставляється в те саме повідомлення, що й «Шукаю…».  
- **/load_all** — підтвердження інлайн-кнопками, прогрес кожні 5 сек., відправка JSON з підписом «Час формування: X хвилин Y секунд».  
- **/analyze** — висновок про корупційні ризики за ПІБ (реєстр з файлу + Gemini); потребує `corrupt_all_data.json` та `GEMINI_API_KEY`.  
- **/revelio** — випадкове ПІБ з реєстру для тесту.
